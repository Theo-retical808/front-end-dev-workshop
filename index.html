Module 1

📄 HTML Basics

The structure of every webpage

What is HTML?

  • HTML = HyperText Markup Language
  • It tells the browser what to show
  • Uses "tags" — instructions wrapped in angle brackets
  • Most tags come in pairs: <tag>content</tag>
🏠 Think of HTML as the blueprint of a house — it defines rooms, doors, and windows, but not the paint color.

Basic Page Structure

<!DOCTYPE html>
<html>
<head>
    <title>My Page</title>
</head>
<body>
    <h1>Hello World!</h1>
    <p>This is my first webpage.</p>
</body>
</html>
🧠 <head> = invisible info | <body> = visible content

Essential HTML Tags

TagWhat it doesExample
<h1>–<h6>Headings (big → small)Page titles
<p>Paragraph of textBody text
<a href="">Clickable linkNavigation
<img src="">ImagePhotos, icons
<ul> <li>Bullet listMenu items
<button>Clickable buttonActions
<div>Container/groupSections
<input>Text fieldForms

Module 2

🎨 CSS Styling

Making your page look amazing

What is CSS?

  • CSS = Cascading Style Sheets
  • Controls colors, fonts, spacing, and layout
  • Separates "how it looks" from "what's on the page"
🎨 If HTML is the skeleton, CSS is the skin, clothes, and makeup!

CSS Syntax

selector {
    property: value;
}

/* Example: Make all h1 headings blue and big */
h1 {
    color: blue;
    font-size: 36px;
}

/* Example: Style elements with class "card" */
.card {
    background-color: white;
    border-radius: 10px;
    padding: 20px;
}
🎯 Selector = WHAT to style | Property = WHICH style | Value = HOW

Selectors: Finding Elements

SelectorTargetsExample
h1All <h1> elementsElement selector
.cardclass="card"Class selector (reusable!)
#headerid="header"ID selector (unique!)
button:hoverWhen mouse hoversPseudo-class
💡 Use classes for most styling. IDs are for unique, one-of-a-kind elements.

Module 3

⚡ JavaScript Interaction

Making your page come alive

What is JavaScript?

  • The programming language of the web
  • Makes pages respond to clicks, typing, scrolling
  • Can change content, styles, and structure dynamically
  • Runs right in the browser — no install needed!
🧠 JavaScript is the brain — it makes decisions and takes actions.

Variables: Storing Data

// "let" = value CAN change
let score = 0;
let name = "Alex";

// "const" = value CANNOT change
const maxLives = 3;

// Types of data:
"hello"       // String (text)
42            // Number
true / false  // Boolean (yes/no)
[1, 2, 3]    // Array (list)
💡 Use let when the value will change, const when it won't.

Talking to HTML (The DOM)

// FIND an element by its ID
const button = document.getElementById("my-button");
const display = document.getElementById("score");

// CHANGE the text inside an element
display.textContent = "Score: 10";

// CHANGE the style
button.style.backgroundColor = "green";

// ADD or REMOVE a CSS class
element.classList.add("hidden");
element.classList.toggle("active");

Events: Responding to Actions

// When the button is CLICKED, run this code:
button.addEventListener("click", function() {
    score = score + 1;
    display.textContent = score;
});

// Other events:
// "input"     → when someone types
// "mouseover" → when mouse hovers
// "keydown"   → when a key is pressed
// "submit"    → when a form is sent
🎮 Events = "When THIS happens, do THAT"

Live Demo: Click Counter

// HTML: <span id="count">0</span>
//       <button id="btn">Click Me</button>

let count = 0;
const countDisplay = document.getElementById("count");
const btn = document.getElementById("btn");

btn.addEventListener("click", function() {
    count = count + 1;
    countDisplay.textContent = count;
});

Flexbox: Smart Layouts

/* Parent container */
.card-grid {
    display: flex;       /* Activate flexbox */
    flex-wrap: wrap;     /* Items move to next line */
    gap: 15px;           /* Space between items */
}

/* Each card */
.card {
    flex: 1 1 300px;     /* Grow, shrink, min-width */
}
Card 1
Card 2
Card 3

Media Queries: Screen-Size Rules

/* Mobile styles (DEFAULT — write these first!) */
.card {
    width: 100%;    /* Full width on phones */
}

/* Tablet and up (768px+) */
@media (min-width: 768px) {
    .card {
        width: 50%;   /* Half width on tablets */
    }
}

/* Desktop (1024px+) */
@media (min-width: 1024px) {
    .card {
        width: 25%;   /* Quarter width on desktop */
    }
}
📐 Mobile-first = start small, add for bigger. Easier to scale UP!

Module 5

🌐 API Integration

Getting data from the internet

What is an API?

  • API = Application Programming Interface
  • A way for your code to talk to other computers
  • You send a request, you get a response
🍽️ Think of a waiter: you tell them what you want (request), they bring it from the kitchen (server), and deliver it to you (response).

fetch() + async/await

// The modern way to get data from the internet

async function getJoke() {
    try {
        // 1. Send request
        const response = await fetch(
            "https://v2.jokeapi.dev/joke/Pun?safe-mode"
        );
        
        // 2. Convert response to usable data
        const data = await response.json();
        
        // 3. Use the data!
        document.getElementById("joke").textContent = data.joke;
        
    } catch (error) {
        // 4. Handle errors gracefully
        console.error("Oops:", error);
    }
}

What JSON Looks Like

// API response (JSON = JavaScript Object Notation)
{
    "type": "single",
    "joke": "Why do programmers prefer dark mode?",
    "id": 232,
    "safe": true
}

// Access with dot notation:
data.joke  → "Why do programmers prefer dark mode?"
data.type  → "single"
data.id    → 232
📦 JSON is just organized data — like a labeled box of information.

Bonus

⚛️ Introduction to React

A sneak peek at modern frontend development

What is React?

  • A JavaScript library made by Facebook (Meta)
  • Used by Instagram, Netflix, Airbnb, Discord, and more
  • Makes building complex UIs easier and faster
  • Key idea: build with reusable components
🧱 Think of LEGO blocks — each component is a block you can reuse and combine to build anything!

Why React? (vs Plain JS)

Plain JavaScript

// Manually update every element
document.getElementById("name")
  .textContent = "Alex";
document.getElementById("score")
  .textContent = score;
document.getElementById("list")
  .innerHTML = items.map(
    i => `<li>${i}</li>`
  ).join("");

😓 You manage every update yourself

React

// Just describe what you want
function App() {
  return (
    <div>
      <h1>{name}</h1>
      <p>Score: {score}</p>
      <ul>
        {items.map(i =>
          <li>{i}</li>
        )}
      </ul>
    </div>
  );
}

✨ React updates the page for you!

Core Concepts

🧩

Components

Reusable UI pieces

Like functions that return HTML

📦

State

Data that changes

When state updates, UI updates

📨

Props

Pass data to children

Like function arguments

A React Component

import { useState } from 'react';

function Counter() {
    // useState = data that can change
    const [count, setCount] = useState(0);

    return (
        <div>
            <h2>Count: {count}</h2>
            <button onClick={() => setCount(count + 1)}>
                Add One
            </button>
        </div>
    );
}
💡 Notice: no document.getElementById needed! React handles the DOM for you.

Getting Started with React
